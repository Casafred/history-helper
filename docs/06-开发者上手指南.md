# 06 - 开发者上手指南

## 1. 环境搭建

### 1.1 系统要求

| 工具 | 最低版本 | 推荐版本 | 安装方式 |
|------|----------|----------|----------|
| Rust | 1.77.2 | 最新稳定版 | `winget install Rustlang.Rustup` |
| Node.js | 18 | 20+ | `winget install OpenJS.NodeJS` |
| npm | 9 | 11+ | 随 Node.js 安装 |
| Git | 2.30 | 最新 | `winget install Git.Git` |

### 1.2 安装步骤

```bash
# 1. 安装 Rust
winget install Rustlang.Rustup
# 安装完成后重启终端，验证：
rustc --version
cargo --version

# 2. 安装 Node.js
winget install OpenJS.NodeJS
# 验证：
node --version
npm --version

# 3. 克隆项目
git clone <repo-url>
cd history-helper

# 4. 安装前端依赖（包含 Tauri CLI）
npm install

# 5. 配置 API 密钥
cp .env.example .env
# 用编辑器打开 .env，填入你的 USPTO_API_KEY
# 申请方式见 docs/02-USPTO-API注册与使用指南.md
```

### 1.3 API Key 获取

1. 访问 https://data.uspto.gov
2. 注册 USPTO.gov 账户（免费，只需有效邮箱）
3. 登录后访问 https://data.uspto.gov/apis/getting-started
4. 按指引生成 API Key
5. 将 API Key 填入项目根目录的 `.env` 文件：

```env
USPTO_API_KEY=你的API密钥
```

> **重要**：`.env` 文件已在 `.gitignore` 中，不会被提交到远程仓库。

## 2. 首次运行

### 2.1 开发模式

```bash
npm run dev
```

这会执行 `cargo tauri dev`，编译 Rust 后端并启动带 WebView 的桌面窗口。

首次编译需要下载并编译所有 Rust 依赖（约 500+ 个 crate），耗时可能 5-15 分钟，取决于网络和机器性能。后续增量编译会快很多。

### 2.2 构建发布版本

```bash
npm run build
```

这会执行 `cargo tauri build`，生成优化后的可执行文件。

产物位于：
- `src-tauri/target/release/history-helper.exe`（可执行文件）
- `src-tauri/target/release/bundle/`（安装包）

## 4. 开发工作流

### 4.1 新增一个 Tauri 命令的完整流程

以新增"查询外国优先权"为例：

**步骤 1：在 `api/uspto.rs` 中添加 API 方法**

```rust
pub async fn get_foreign_priority(
    &self,
    app_number: &str,
) -> Result<PatentApplicationResponse, UsptoApiError> {
    let url = format!("{}/{}/foreign-priority", USPTO_API_BASE, app_number);
    let resp = self.get_with_retry(&url).await?;
    let data = resp.json::<PatentApplicationResponse>().await?;
    Ok(data)
}
```

**步骤 2：在 `lib.rs` 中注册命令**

```rust
#[tauri::command]
async fn fetch_foreign_priority(
    app_number: String,
    state: tauri::State<'_, AppState>,
) -> CommandResult {
    let client = match get_or_create_client(&state) {
        Ok(c) => c,
        Err(e) => return CommandResult::err(e),
    };
    let normalized = match normalize_us_application_number(&app_number) {
        Ok(n) => n,
        Err(e) => return CommandResult::err(e.to_string()),
    };
    match client.get_foreign_priority(&normalized).await {
        Ok(data) => CommandResult::ok(serde_json::to_value(data).unwrap_or_default()),
        Err(e) => CommandResult::err(e.to_string()),
    }
}
```

**步骤 3：在 `lib.rs` 的 `invoke_handler` 中注册**

```rust
.invoke_handler(tauri::generate_handler![
    // ... 已有命令
    fetch_foreign_priority,  // 新增
])
```

**步骤 4：在前端 `app.js` 中调用**

```javascript
const result = await invoke("fetch_foreign_priority", { appNumber: input });
if (result.success && result.data) {
    renderForeignPriority(result.data);
}
```

## 5. 调试方法

### 5.1 Rust 后端日志

开发模式下，`tauri-plugin-log` 会输出日志到控制台：

```rust
log::info!("Fetching application: {}", app_number);
log::warn!("Rate limited, retrying...");
log::error!("API error: {}", e);
```

### 5.2 前端调试

Tauri 开发模式下，右键窗口 → "检查元素"可打开 DevTools，支持：
- Console 日志
- Network 请求
- DOM 检查
- JavaScript 断点

### 5.3 API 调试

可用 curl 直接测试 USPTO API：

```bash
curl -X "GET" \
  "https://api.uspto.gov/api/v1/patent/applications/14412875" \
  -H "X-API-KEY: YOUR_API_KEY"
```

### 5.4 常见问题排查

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 编译报错 `zmij` 相关 | 项目路径含中文字符 | 将项目移至纯 ASCII 路径 |
| API 返回 403 | API Key 无效或缺失 | 检查 .env 文件中 USPTO_API_KEY |
| API 返回 429 | 请求频率超限 | 增大 REQUEST_INTERVAL_MS |
| 窗口空白 | 前端资源路径错误 | 检查 tauri.conf.json 中 frontendDist |
| invoke is not a function | Tauri v2 API 路径错误 | 使用 `window.__TAURI__.core.invoke` |

## 6. 代码质量检查

```bash
# 格式检查
cargo fmt --check

# 自动格式化
cargo fmt

# 静态分析（Clippy）
cargo clippy -- -D warnings

# 运行测试
cargo test

# 完整检查流程
cargo fmt && cargo clippy -- -D warnings && cargo test
```

## 7. Git 工作流

```bash
# 创建功能分支
git checkout -b feat/foreign-priority dev

# 开发完成后提交
git add .
git commit -m "feat(api): add foreign priority query"
git push origin feat/foreign-priority

# 合并到 dev
git checkout dev
git merge feat/foreign-priority
git push origin dev
```

提交信息格式：`<type>(<scope>): <subject>`
