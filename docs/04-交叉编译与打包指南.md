# 04 - 交叉编译与打包指南

## 1. 环境准备

### 1.1 Windows 开发环境（当前）

```powershell
# 安装 Rust（如已安装可跳过）
winget install Rustlang.Rustup

# 安装 Tauri CLI
cargo install tauri-cli

# 安装前端工具链
npm install
```

### 1.2 Linux 交叉编译环境

在 Linux 上交叉编译 Windows 可执行程序需要以下步骤：

```bash
# 安装 Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 添加 Windows GNU 目标链
rustup target add x86_64-pc-windows-gnu

# 安装 MinGW 交叉编译工具链
# Ubuntu/Debian:
sudo apt install mingw-w64

# Arch Linux:
sudo pacman -S mingw-w64-gcc

# Fedora:
sudo dnf install mingw64-gcc
```

## 2. 使用 cross 工具（推荐）

`cross` 是 Rust 社区推荐的交叉编译工具，使用 Docker 容器提供完整的交叉编译环境，无需手动配置工具链。

### 2.1 安装 cross

```bash
cargo install cross --git https://github.com/cross-rs/cross
```

### 2.2 确保 Docker 已安装并运行

```bash
# 安装 Docker
# Ubuntu/Debian:
sudo apt install docker.io
sudo systemctl start docker
sudo usermod -aG docker $USER

# 验证 Docker 运行
docker ps
```

### 2.3 交叉编译

```bash
# 编译 Windows x64 目标
cross build --release --target x86_64-pc-windows-gnu

# 编译产物位于
# target/x86_64-pc-windows-gnu/release/history-helper.exe
```

### 2.4 优势

- 无需手动安装 MinGW 工具链
- 自动处理 C 依赖的交叉编译
- 环境一致性好，避免工具链版本不匹配

## 3. 手动交叉编译（备选方案）

如果不使用 Docker，可手动配置 MinGW 工具链：

### 3.1 配置 Cargo

在项目根目录创建 `.cargo/config.toml`：

```toml
[target.x86_64-pc-windows-gnu]
linker = "x86_64-w64-mingw32-gcc"
ar = "x86_64-w64-mingw32-ar"
runner = "wine"  # 可选：用于在 Linux 上测试 Windows 程序
```

### 3.2 编译

```bash
cargo build --release --target x86_64-pc-windows-gnu
```

## 4. Tauri 应用打包

### 4.1 Windows 平台打包（在 Windows 上）

```powershell
# 开发模式
cargo tauri dev

# 构建发布版本
cargo tauri build

# 产物位于
# src-tauri/target/release/bundle/msi/
# src-tauri/target/release/bundle/nsis/
```

### 4.2 交叉编译注意事项

Tauri 应用的交叉编译比纯 Rust 项目更复杂，因为涉及 WebView2 等系统依赖。

**推荐方案**：

1. **CI/CD 方式**：使用 GitHub Actions 在 Windows Runner 上构建
2. **直接在 Windows 上构建**：最简单可靠
3. **Linux 交叉编译**：仅编译 Rust 后端，前端资源预构建

### 4.3 GitHub Actions 自动构建（推荐）

创建 `.github/workflows/build.yml`：

```yaml
name: Build
on:
  push:
    tags:
      - 'v*'

jobs:
  build-windows:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm install
      - run: cargo tauri build
      - uses: actions/upload-artifact@v4
        with:
          name: windows-build
          path: src-tauri/target/release/bundle/
```

## 5. 打包前检查清单

打包前**必须**执行以下检查：

```bash
# 1. 拉取最新代码
git pull origin main

# 2. 确认无未提交变更
git status

# 3. 确认本地与远程一致
git diff origin/main

# 4. 确认版本标签
git tag -l | grep v0.1.0

# 5. 运行格式检查
cargo fmt --check

# 6. 运行静态分析
cargo clippy -- -D warnings

# 7. 运行测试
cargo test

# 8. 构建
cargo tauri build
```

## 7. 常见问题

### Q: 交叉编译时链接错误怎么办？

A: 确保安装了正确的 MinGW 工具链，且 `.cargo/config.toml` 中的 linker 和 ar 路径正确。推荐使用 `cross` 工具避免此类问题。

### Q: Tauri 交叉编译支持哪些目标？

A: Tauri 官方推荐在目标平台上原生构建。交叉编译主要适用于纯 Rust 后端。对于 Tauri 应用，推荐使用 GitHub Actions 在对应平台的 Runner 上构建。

### Q: 打包体积过大怎么办？

A: 检查以下几点：
1. 确认是 release 模式构建（`--release`）
2. Cargo.toml 中启用 LTO 和优化体积设置
3. 前端资源压缩
4. 移除不必要的依赖

```toml
[profile.release]
lto = true
codegen-units = 1
strip = true
opt-level = "s"
```
